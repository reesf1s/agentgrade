# AgentGrade Model Training and Privacy

## Executive Summary

AgentGrade today uses a three-layer scoring stack:

1. A rubric-based LLM judge for claim analysis, scoring, and interventions
2. Deterministic guardrails for trust, cost control, and false-positive protection
3. A learned calibration model trained from human labels

This means the product is already more than a prompt wrapper, but it is not yet a fine-tuned end-to-end proprietary foundation model.

## What Is Learned Today

The learned component is a supervised calibration model.

It learns from:
- human overrides
- pasted gold-set examples
- labels on real production conversations
- labels on synthetic regression examples

It adjusts:
- overall score
- accuracy
- hallucination
- resolution
- tone
- sentiment
- escalation handling

It does not directly rewrite the base model weights.

## Privacy Model

AgentGrade supports two learning scopes:

### Workspace Private

Use labels only inside that customer workspace.

Use cases:
- org-specific policy interpretation
- company-specific tolerance for escalation
- domain-specific scoring quirks

### Global Anonymous

Use anonymized scorer features and labels across customers.

This path does **not** export raw transcript text into the shared calibration model.

Examples of sharable signals:
- current scorer outputs
- confidence level
- claim counts
- flags
- hard-fail status
- human labels
- example type and task metadata

This lets AgentGrade improve cross-customer calibration while protecting client content.

## What Is Not Yet True

These statements would be misleading today:
- "AgentGrade is fully fine-tuning its own scoring foundation model in production"
- "All customer transcripts are pooled into a single global training corpus"
- "Every score is generated by a custom proprietary model"

Those are future states, not current production truth.

## Why This Is Still Valuable

This architecture is already strong enough to matter commercially because it provides:
- explainable rubric outputs
- auditable human correction loops
- lower marginal scoring cost through deterministic fast paths
- org-specific adaptation without dangerous data leakage
- a clean path to a proprietary scorer over time

## Path to a Proprietary Scoring Model

The practical roadmap is:

1. Build and maintain a large trusted gold set
2. Measure inter-rater agreement and label quality
3. Export anonymized supervised training sets
4. Train internal regression or ranking models for score prediction and confidence estimation
5. Only then evaluate provider fine-tuning or a proprietary end-to-end scoring model

This keeps the product honest today while still making the long-term moat very real.
